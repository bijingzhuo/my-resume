# 个人简历网站

## 项目目标
- 我是需要一个长期可维护的个人简历网站
- 核心原则：内容和代码分离，加新内容只改 Markdown，不动代码
- 风格：简洁、专业、加载快

## 技术栈
- Astro（静态站点生成）
- Markdown（内容存储）
- 纯 CSS（不引入 UI 框架，保持轻量）
- 部署到 GitHub Pages

## 目录约定
- 所有简历内容放在 src/content/ 下，按类别分文件夹
- 每个项目/经历是一个独立的 .md 文件，带 frontmatter
- 不要在 .astro 文件里硬编码任何简历内容

## 代码约定
- 优先使用 Astro 原生功能，避免引入不必要的依赖
- 组件保持小而专注
- 注释用中文

## 我的偏好
- 我在 vibe coding 学习阶段，请在做较大改动前简要说明思路
- 遇到多种实现方案时，列出选项让我选，不要替我决定架构
- 每完成一个独立功能停下来，方便我 git commit
- 解释代码时讲清楚为什么这么写，不只是写了什么
